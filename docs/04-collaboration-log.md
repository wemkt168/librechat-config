## 协作流程总结

### 标准流程
1. **我（AI 助手）**：写完整代码，提供标准化指令
2. **你（用户）**：转发指令给对应的 AI IDE
3. **AI IDE**：执行任务，测试，反馈结果
4. **我（AI 助手）**：确认结果，继续下一步

### 关键原则
- **代码质量**：由我控制（完整、正确、有注释）
- **执行准确性**：由 AI IDE 控制（按步骤执行、测试、反馈）
- **效率**：减少来回沟通，一次性完成

### ⚠️ 任务状态确认规则（重要！）

**当给出指令给其他 AI IDE（如 GitHub Copilot、Zeabur AI IDE）时：**

1. **禁止假设任务已完成**：
   - 如果用户没有明确回复"已完成"或提供反馈，不能假设任务已完成
   - 不能在没有确认的情况下，将任务标记为"已完成"或"进行中"

2. **任务状态管理**：
   - 给出指令后，任务状态应该是"待执行"或"等待反馈"
   - 只有收到用户明确的完成反馈后，才能标记为"已完成"

3. **状态确认方式**：
   - 等待用户明确回复执行结果
   - 如果用户提供 API KEY，可以通过 API 调阅和检查状态
   - 如果没有 API KEY，必须等待用户反馈

4. **示例场景**：
   - 给出 GitHub 部署指令 → 标记为"等待执行"，直到用户反馈
   - 给出 Zeabur 部署指令 → 标记为"等待执行"，直到用户反馈
   - 用户回复"已完成" → 才能标记为"已完成"
